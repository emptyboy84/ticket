# 📚 오늘의 티켓팅 웹사이트 제작 핵심 노트

> 이 문서는 집에서도 코드를 복습하며 **"아무것도 보지 않고 스스로 만들기"** 목표를 달성할 수 있도록 오늘 안티그라비티(AI)와 나눈 핵심 대화를 요약한 비법 노트입니다.

---

## 1. 프론트엔드와 백엔드의 완벽한 분리 
넥스트제이애스(Next.js) 폴더 구조를 보며 깨달은 가장 중요한 개념입니다.

*   **🖥️ 프론트엔드 (사용자 화면)**
    *   `src/components/`: 버튼, 카드 모양 등을 찍어내는 부품 공장 (예: `SeatButton.tsx`)
    *   `app/page.tsx`: 부품들을 모아 조립하고, 백엔드에 배달(fetch)을 시키는 메인 쇼윈도
    *   `app/globals.css`: 화면을 꾸미는 물감통 (Tailwind CSS)
*   **⚙️ 백엔드 (서버 & 데이터베이스 연동)**
    *   `app/api/`: 프론트엔드의 요청을 받는 진짜 백엔드 접수처 (`route.ts`)
    *   `src/lib/mongodb.ts`: DB(MongoDB)와 연결해주는 안전한 수도관
    *   `src/models/`: 데이터의 생김새(뼈대)를 정해주는 규칙서 (예: `Movie.ts`)

---

## 2. 혼자서 사이트 만들기 5단계 로드맵

백지 상태에서 시작할 때 이 순서를 머릿속에 떠올리면서 타이핑하세요!

1.  **파이프 깔기 (DB 연동)**: `.env.local`에 DB 주소를 넣고, `lib/mongodb.ts`에서 수도관이 중복으로 생기지 않도록 방어 코드(`mongoose.connection`)를 짭니다.
2.  **데이터 규칙 정하기 (모델링)**: `src/models/`에 영화(`Movie`)와 예약(`Booking`) 등의 엑셀 표 형태를 정의합니다.
3.  **데이터 창구 열기 (API 만들기)**: `app/api/...` 경로에 GET(데이터 줘!), POST(데이터 저장해!) 함수를 만듭니다.
4.  **UI 조각 만들기**: 좌석 버튼(`SeatButton`)처럼 반복해서 쓸 화면 부품을 `src/components`에 예쁘게 디자인해서 만들어 둡니다.
5.  **생명 불어넣기 (조립/통신)**: `app/page.tsx`에 `useState`, `useEffect`를 써서 처음 로딩 때 영화 정보를 부르고, 클릭 시 API(POST)로 데이터를 넘깁니다.

## 3. 🚨 오늘 마주친 무서운 에러들과 해결법

개발 중에 겪었던 무서운 에러들, 알고 보면 별것 아닙니다. 집에서 또 마주치더라도 당황하지 마세요!

*   **에러 1: Turbopack(터보팩) Rust Panic 뻗음 현상 (`tokio-runtime-worker panicked`)**
    *   **원인**: Next.js가 서버를 빨리 돌리려다 캐시(찌꺼기)가 꼬인 버그입니다.
    *   **해결**: 서버를 끄고 터미널에서 `rmdir /s /q .next` 를 입력해 숨겨진 `.next` 폴더를 날려버린 뒤 다시 `npm run dev`를 켭니다.
*   **에러 2: 몽고DB 비밀번호에 특수문자(`$`)가 들어갈 때**
    *   **원인**: `Pa$$w0rd` 처럼 특수기호가 URI 주소에 그대로 들어가면 컴퓨터가 주소로 인식하지 못하고 깨집니다.
    *   **해결**: `$` 문자는 웹 인코딩 규칙에 따라 `%24` 형식으로 변환해서 입력해야 정상 연결됩니다.
*   **에러 3: 내 로컬에선 잘 되는데 Vercel 배포 시 `npm run build` 에러**
    *   **원인**: Windows는 파일명의 대소문자(예: `seatButton` ➔ `SeatButton`)가 바뀌어도 Git이 눈치를 못 채서, 쌩소문자 파일이 Vercel 서버로 올라갔기 때문입니다. (Linux 기반 서버는 대소문자를 엄격하게 가립니다)
    *   **해결**: 터미널에 `git rm -r --cached .` ➔ `git add .` ➔ `git commit -m "대소문자 수정"` 순서대로 쳐서 깃을 강제로 새로고침 해주고 올립니다.
*   **에러 4: 화면 디자인이 안 먹고 텍스트만 흰 바탕에 덜렁 나올 때**
    *   **원인**: `Tailwind CSS` 설정이 꼬였기 때문입니다.
    *   **해결**: `postcss.config.mjs`의 꼬인 설정을 표준으로 리셋하고, 찌꺼기(`.next` 폴더)를 지우면 해결됩니다.
